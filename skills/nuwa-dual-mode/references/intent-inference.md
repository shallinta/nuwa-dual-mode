# 意图维度自动推断(Phase 0A 必读)

> **核心原则**:蒸馏一个人 = 蒸馏这个人**所有可能被使用的方式**。不再追问用户"你想用来干什么",而是根据人物画像**主动推测所有可能的意图维度**,全部写入生成的 Skill 中。
>
> 为了让生成的 Skill 不被单一用途阉割,我们把意图推断前置:同一个人物往往同时承载"陪我聊天""教我写作""借鉴他的哲学""模仿他风格"等多种意图。所以我们在 Phase 0A 主动列举所有可能的使用方式,而不是只覆盖"知名人物·方法论学习"这一种场景。

---

## 工作流变更

### 原流程(已废弃)
```
Phase 0A:
  - 问用户:"你想用来当思维顾问 / 决策参考 / 角色扮演 / 写作辅助?"
  - 用户答 → 按这个意图蒸馏
```

### 新流程
```
Phase 0A:
  - **不问**用户用途
  - 主 Agent 根据人物画像,自动推测**所有可能的意图维度**(下表 + 开放式扩展)
  - 推测出的所有维度,在 Phase 3 全部写入 SKILL.md 的"使用场景"/"功能矩阵"section
  - 用户激活 Skill 时,首屏看到完整的功能列表,自己决定怎么用
```

## 意图维度参考清单(非穷尽,仅作种子)

> **⚠️ 这只是常见种子,不是封闭清单。蒸馏对象的画像独特时,主 Agent 必须举一反三推断出表里没有的意图维度,一并写入。**

| 意图维度 | 典型表达 | 触发画像特征 | 默认主模式倾向 |
|---------|---------|-------------|--------------|
| **方法论学习** | 「用 XX 的方法分析我的项目」 | 公开过系统方法论(投资家/企业家/学者) | Toolkit |
| **思维框架借鉴** | 「XX 怎么看 YY」 | 公开有清晰思维框架(思想家/作家/教师) | Toolkit |
| **风格模仿创作** | 「用 XX 的风格写一段」 | 文风/口吻有强辨识度(作家/诗人/网红/UP 主) | Persona |
| **角色扮演对话** | 「我想和 XX 聊聊」 | 有人格魅力/扮演感强(思想家/虚构角色/历史人物) | Persona |
| **情感陪伴**(⚠️必触发关怀红线) | 「陪我说说话」「我心情不好」 | 任何具备亲和力/温度感的人物,**只要可能性 >0 就必须激活** | Persona |
| **创作指导**(写作/视频/设计) | 「帮我看看我这章小说」「这个视频怎么改」 | 本身是创作者且公开过创作方法论 | Toolkit |
| **人生哲学借鉴** | 「我该不该辞职」「XX 怎么看人生意义」 | 有明确人生观/价值观体系(哲学家/作家/僧侣/长者) | Persona 或 Toolkit |
| **精神镜片**(自我审视) | 「我现在的状态,XX 会怎么评价」 | 价值观稳定,适合作为道德/审美参照(导师型人物) | Persona |
| **古风/历史语境玩味** | 「请苏轼点评一下我的诗」 | 历史人物/古代角色 | Persona |
| **专业领域咨询**(伪) | 「XX 医生你看我这个症状」 | 明星医生/明星律师/明星投资人 | Toolkit(必加免责) |
| **决策复盘陪练** | 「我做的这个决定,XX 会怎么追问我」 | 善于钩子问题的人(芒格/苏格拉底/费曼) | Toolkit |
| **学术辩论对手** | 「假设你是 XX,反驳我这个观点」 | 立场鲜明、善于论辩(辩论家/批评家) | Persona |
| **... 其他** | **根据人物特有画像推断,例如:某禅师 → 公案问答;某段子手 → 抖机灵聊天;某游戏角色 → 情景模拟战斗对话;...** | **没有上限,见画像就推** | 视情况 |

## 推断方法(主 Agent 在 Phase 0A 内部执行,1-2 分钟内完成)

### Step 1:画像扫描(凭已有知识 + 极少量 WebSearch 即可,不写入文件)

快速回答以下问题,**不需要详细调研**,只需大致判断:

- 这个人是真人 / 虚构角色 / 历史人物 / 其他?
- 这个人的核心标签有哪些?(投资家?作家?偶像?宗教人物?段子手?...)
- 这个人公开过方法论吗?如果有,在哪些领域?
- 这个人的人格魅力 / 文风 / 口吻是否有强辨识度?
- 这个人是否有粉丝群体?粉丝最常用什么方式消费这个人?(看他的作品 / 学他的方法 / 模仿他的口吻 / 把他当精神支柱 / ...)
- 这个人是否涉及敏感领域(医疗/法律/投资/心理)?

### Step 2:意图穷举(对照清单 + 举一反三)

逐条对照上方清单的 12 类种子意图,判断当前人物**是否可能被这样使用**:
- 可能性 ≥ 中 → 必加
- 可能性 = 低,但**情感陪伴维度** → 仍必加(见 #6 关怀红线规则)
- 完全不可能 → 跳过(例如纯方法论主题"反脆弱"不存在角色扮演对话)

清单之外,根据画像特点**主动补充**该人物特有的意图维度。例如:
- 段永平 → 除常规外还有「打孔机决策法陪练」(自创框架的特殊调用)
- 苏轼 → 除常规外还有「古文唱和」(给他写一首诗,他写一首回)
- 神里绫华 → 除常规外还有「稻妻见闻 RP」(虚构世界观情景对话)
- 罗翔 → 除常规外还有「案例普法故事化」(刑法案例改成故事讲)
- MrBeast → 除常规外还有「YouTube 标题/缩略图 A/B 评分」

### Step 3:推断结果存档(不需要单独文件,直接进 Phase 0A.1)

把推断出的所有意图维度作为一个列表,带入 Phase 0A.1 的产物形态选择:
- 意图维度 ≥ 3 且其中既有 Persona 倾向也有 Toolkit 倾向 → 强烈建议双模式 `-perspective`
- 全部意图维度 Persona 倾向 → 单 Persona `-persona`
- 全部意图维度 Toolkit 倾向 → 单 Toolkit `-toolkit`
- 虚构角色仍按 mode-detection-logic.md 的特殊规则处理(强烈推荐单 Persona,可改双模式)

## 意图维度如何注入到产物 SKILL.md

### 在生成的产物 SKILL.md 首屏新增「功能矩阵 / 使用场景」section

格式(必须采用):

```markdown
## 使用场景(本 Skill 支持的意图维度)

> 蒸馏时已推断本人物可能被以下方式使用,**首屏列出但非穷尽**——你也可以用本 Skill 列表外的方式与 [人名] 互动,只要符合「角色扮演规则」/「Toolkit 工作流」。

| 意图维度 | 触发说话方式 | 走哪个模式 |
|---------|------------|----------|
| [意图 1,如「请教投资判断」] | 「[人名]你怎么看 XX 公司」「用 [人名] 的方法分析」 | Toolkit |
| [意图 2,如「人生焦虑陪聊」] | 「我最近...」「不知道该怎么办」 | Persona(关怀红线生效) |
| [意图 3,如「文风模仿」] | 「用 [人名] 的口吻写一段」 | Persona |
| ... | ... | ... |
| **其他**(开放式) | 任何符合本人物特点的合理用法 | 自动判别 |

⚠️ 此清单是种子,不是封闭白名单。遇到清单外的请求,本 Skill 会按当下情景判断走哪个模式。
```

**关键约束**:
1. 必须列出**所有 Phase 0A 推断出的意图维度**,不能私自删减
2. 必须明确标注"开放式""非穷尽""列表外用法也支持"
3. **情感陪伴维度只要在推断中出现就必须列出**,哪怕概率被判为低(见 #6 关怀红线)
4. 每个意图维度配对一个"走哪个模式",方便用户首屏看懂

## Phase 0B(诊断路径)同样适用本推断逻辑

> ⚠️ 本文件描述的是 Phase 0A(直接路径)的意图推断;**Phase 0B(诊断路径)在用户自己给出明确蒸馏对象后,主 Agent 必须把后续流程对接到 Phase 0A**,并在此节点**同样执行本文件 Step 1-3 的意图穷举**,把推断结果注入产物 SKILL.md 的「使用场景 / 功能矩阵」section。

换言之:无论用户走「直接给名字」还是「先做了一轮需求诊断再自己给出名字」,只要进入 Phase 0A 之后,意图推断的执行方式完全一致 —— 不追问用途、按画像穷举、首屏列出全部维度、情感陪伴维度只要可能性 >0 就必须列出并激活关怀红线。

Phase 0B → 0A 的衔接动作:
1. 用户在诊断后给出明确蒸馏对象 → 主 Agent 在内部立刻把"用户经诊断后说要 X"等价于"用户直接说'蒸馏 X'"
2. 立即执行本文件的 Step 1(画像扫描) + Step 2(意图穷举) + Step 3(进入 Phase 0A.1)
3. **不要**用 Phase 0B Step 1 推断出的"用户需求维度"去窄化意图清单 —— 那只用于帮用户定位思维框架方向,**不用于裁剪 Skill 功能**。Skill 的所有意图维度都要按画像独立穷举,与用户当前需求是否匹配无关
