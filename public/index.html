蒸馏思维操作系统

不是话术模仿。提取一个人的怎么想、怎么判断、怎么说话、什么不做。心智模型分两栏：价值观镜片 + 操作招式。

🎯

意图自动推断，不追问用户

女娲根据人物画像自动推断全部可能意图维度——方法论学习、风格模仿、角色扮演、决策复盘……不问你"想用来干嘛"。

🔒

关怀红线硬门槛

情感陪伴可能性 >0 时，产物必含中/英/新/港/台五地心理热线。Persona 遇到危机信号会暂时出戏给真实求助资源。

⚡

6 个 Agent 同时开工

著作、对话、表达DNA、他者视角、决策案例、时间线——六个维度并行调研，五分钟覆盖单人两小时的调研量。

🔄

增量更新，不重跑全流程

已蒸馏的 skill 支持增量更新——只拉最新动态。三选一：覆盖重做 / 增量更新（推荐） / 新建 v2 并存。

🌐

跨 50+ Runtime 通用

基于开放的 Agent Skills 协议，Claude Code、Codex、Cursor、OpenClaw、Hermes、Gemini CLI 等均可运行。

蒸馏流水线

从输入到交付，9 个步骤自动流转——红色标记为显式检查点，必须用户确认才继续

🔍
Phase 0 · 检查点 ✓
入口分流 + 已有 Skill 检测
识别蒸馏对象的人物画像、活动地域、主要语言。扫描本地是否已有同名 skill——有则触发三选一：覆盖重做 / 增量更新 / 新建 v2 并存。
画像扫描语言判定更新检测
🎯
Phase 0A · 检查点 ✓
意图维度自动推断 + 形态选择
根据人物画像自动推测 12+ 种意图维度（方法论学习 / 风格模仿 / 角色扮演 / 情感陪伴 / 决策复盘……），举一反三补充独有维度。自动判定产物形态（双/单模式），交给用户确认——首次出现术语时先用白话解释。
12+ 意图自动判定术语解释
⚡
Phase 1
6 个 Agent 并行调研
著作文献 / 对话访谈 / 表达 DNA / 他者评价 / 决策案例 / 时间线——六个维度同时开工，按人物语言地域自动切换信息源白名单。一手来源优先，知乎/公众号/百度百科永久排除。
并行调研信息源过滤双语策略
📊
Phase 1.5 · 检查点 ✓
调研质量摘要
生成一手/二手来源占比、跨 Agent 矛盾点检测、信息不足维度标注。总来源数 < 10 时建议降低期望或补充调研。
质量审计矛盾检测来源占比
🧠
Phase 2
心智模型框架提炼
三重验证提取核心思维模型。分两栏：Part A 价值观镜片（2-5 个）——跨域复现/生成力/排他性；Part B 操作招式（1-5 个）——此人亲口命名的可操作清单。同人与官方冲突时暂停询问。
三重验证两栏分拆同人协商
🏗️
Phase 3
Skill 构建与注入
按模板生成三层目录（shared / mode-persona / mode-toolkit）。注入功能矩阵、关怀红线（5 地区热线 + 兜底）、时代锚定声明、分级免责、时效性声明、Agentic Protocol。
模板填充关怀红线时效声明
🔬
Phase 4 · 检查点 ✓
质量验证 + 修复循环
按推断意图选择验证套件。Toolkit 招式枚举校验（亲口招式全部召回，漏一个 = FAIL）。关怀红线三类信号模拟（普通低落/极端情绪/不健康依赖）全过才算 PASS。最多 2 轮修复，第 2 轮仍 FAIL 必须用户确认是否交付。
招式校验关怀测试2轮上限
✨
Phase 5
双 Agent 精炼
两个独立 Agent 分别审查：意图维度覆盖齐全度 + 强制 section 注入完整度。交叉验证后合并建议，确保产物没有遗漏。
交叉验证意图齐全section 完整
🚀
交付
打包注册 + 五段收尾
产物上传注册为 custom skill。主消息输出五段话：产物形态 / 使用方式（含切换模式与典型问法）/ 局限提醒 / 后续维护（修改/更新/卸载）/ Skill 简介与女娲引导。不可省略。
打包注册五段收尾维护说明
