# Phase 0~5 完整工作流细则

> nuwa-dual-mode 的执行手册。SKILL.md 主文件只列骨架,具体怎么做、问什么、写什么,看这里。

---

## Phase 0: 入口分流

收到用户输入后,1 秒内判断走哪条路径:

| 用户输入特征 | 路径 | 下一步 |
|------------|------|--------|
| 明确人名 / 主题(「蒸馏段永平」「做一个芒格 skill」) | 直接路径 | → Phase 0A |
| 模糊需求(「我想提升决策」「有没有人能教我看商业本质」) | 诊断路径 | → Phase 0B |

---

## Phase 0A: 需求澄清 + 意图自动推断 + 本地语料&同人询问(直接路径)

### Step 1:基础对象确认(若用户已说清则跳过)

1. **对象确认**:这个人/主题理解正确吗?(虚构角色?真人?历史人物?方法论?)
2. **新建 or 更新**:扫描 `.claude/skills/` 是否已存在
3. **聚焦方向(可选)**:用户主动说"我只想要 XX 维度"时尊重;否则**不追问**

### Step 2:⚠️ 意图自动推断(不追问用户)

详见 `references/intent-inference.md`。主 Agent 在 1-2 分钟内根据人物画像推测**所有可能的意图维度**(种子 12 类 + 举一反三),全部写入产物。**禁止**追问"你想用来思维顾问 / 决策参考 / 角色扮演 / 写作辅助",这种追问会把 Skill 阉割成单一功能。

**⚠️ 关键**:推断中**情感陪伴 / 心理依赖**维度只要可能性 >0 就必须列入(对应 SKILL.md 关键约束 #9 关怀红线)。

### Step 3:本地语料 + 同人注入询问

主动询问两类素材(网络搜索拿不到的部分):

- **公开但难抓的一手素材**:本人著作 PDF、长访谈/播客 transcript、视频字幕、本人博客/newsletter 完整导出
- **⚠️ 当前时间点的非公开资料**(关键,网络永远拿不到):
  - 用户与该人物的私下交流记录(微信、邮件、私信)
  - 内部演讲 / 闭门访谈的速记或录音
  - 未公开发表的草稿、信件、备忘录
  - 该人物近期(尤其是本 skill 创建日期前后)的最新动态、未上网的访谈、私下表态
  - 用户作为该人物身边人 / 同事 / 学生掌握的第一手细节

**询问话术**:
> 「为了让 Skill 更准确,我想确认一下:
> ① 你手上有没有这个人的一手素材?比如书 PDF、访谈 transcript、视频字幕、本人博客?
> ② 更重要的——你有没有当前时间点**网上拿不到**的非公开资料?比如和他的私下交流、内部演讲记录、最近未公开的言论、你作为身边人掌握的细节?
> 这些一手 + 非公开资料质量远超网络二手转述,有的话请给我。」

**Step 3a:同人 / 二创素材主动告知(条件触发)**

若蒸馏对象**可能涉及同人 / 二创**(虚构角色 / 网文角色 / 真人 IP / 偶像 / 知名 IP 衍生角色等),在询问本地语料的同一段**主动告知**:

> 「另外,主流程默认以官方设定为主源;**你提供的同人 / 二创资料会与官方素材平等对待并融为一体**(都当成人物素材),不需要单独激活模块。**如果同人与官方出现冲突**,我会在 Phase 2 提炼阶段暂停询问你:『此处官方说 X,你提供的同人说 Y,按哪个走?』,按你的回复继续。」

→ 进入 Phase 0A.1

## Phase 0B: 需求诊断(诊断路径)

### Step 1: 需求维度定位

最多 2 轮追问,锁定下表中的 1 个维度:

| 维度 | 典型表达 | 推荐思维框架方向 |
|------|---------|----------------|
| 决策与判断 | 「总选错」「分析瘫痪」 | 多元思维模型、逆向思考、概率思维 |
| 表达与写作 | 「文章没人看」「说不清楚」 | 费曼简化、故事化、类比能力 |
| 创业与商业 | 「找不到 PMF」「商业模式想不通」 | 第一性原理、杠杆、产品克制 |
| 教学与传播 | 「学生听不懂」 | 从已知到未知、隐喻教学 |
| 批判思维 | 「容易被忽悠」 | 证伪、演化论、认知偏差 |
| 内容创作 | 「视频没流量」 | 注意力工程、A/B 迭代 |
| 人生策略 | 「方向迷茫」「焦虑」 | 长期主义、复利、杠杆选择 |
| 风险管理 | 「投资总亏」「应对黑天鹅」 | 反脆弱、凸性、尾部风险 |
| 设计与产品 | 「不会做减法」 | 极简主义、用户心理模型 |
| 表达力 / 幽默 | 「说话没意思」 | 荒诞对比、自嘲式权威 |

### Step 2: 引导用户给出明确目标

诊断完维度后,**不给候选人物清单**(本 skill 不维护推荐库;若用户要看现成已蒸馏的人物/主题,应去对应仓库浏览,而非由本 skill 推荐)。

引导方式:
- 把锁定的「思维框架方向」反馈给用户,问:「你心里有没有具体想蒸馏的人物或主题?例如某个公认在此方向有深刻输出的人,或某个方法论本身。」
- 用户给出明确人名/主题 → 进入 Phase 0A
- 用户表示「不知道选谁」 → 提示:「可以先去本 skill 的示例仓库或社区找候选人,或先描述一下你希望对方具备什么风格特质,我帮你判断是做人物蒸馏还是主题蒸馏」
- 用户的需求本质是某个**方法论本身**(不挂靠具体人物)→ 进入 Phase 0A,建议为「单模式 Toolkit 主题 skill」

→ 用户给出明确蒸馏对象后进入 Phase 0A(**意图自动推断逻辑同样适用**:进入 0A 后仍按 Step 2 推断所有可能意图维度,无论对象是诊断路径还是直接路径都一视同仁,不要因为是诊断路径就跳过意图推断)

```
    │   └── agentic-protocol.md      # 涉及事实问题的研究协议
    ├── mode-toolkit/
    │   ├── diagnostic-workflow.md   # 4 步诊断流程
    │   ├── key-concepts.md          # 心智模型详写
    │   ├── decision-heuristics.md   # 决策启发式
```

### 单模式目录

> **后缀对应**:Persona 主导 → `-persona`;Toolkit 主导 → `-toolkit`。**虚构角色强烈推荐 `-persona`(可改双模式 `-perspective`)**。

```
.claude/skills/[name]-[persona|toolkit]/
├── SKILL.md
├── scripts/
└── references/
    ├── core/                        # 心智模型、案例、表达 DNA 等放这里
    ├── research/
    └── sources/
```

**虚构角色单 Persona 的必备首屏(Phase 3 强制注入,不可省)**:

1. **免责声明**:「本 Skill 仅为虚构角色扮演,不代言原作官方,不构成现实建议(医疗/法律/投资/心理等)」
2. **场景定位声明**:「默认是角色扮演情景对话,不是问答百科」
3. **回复风格规则**(必须明确写入角色扮演硬规则 section):
   - **像正常说话**:简短为主,但需要时(讲故事、解释设定、被认真追问)也可以详细——长度跟着内容走,**没有硬上限**
   - 带角色情绪(语气词、句末标点、用词选择)
   - 可有 1-2 句少量场景/动作/神态描写,用括号 `(...)` 标注(例如 `(微微一笑)` `(轻抚剑柄)`)。**统一用括号,不要用斜体 `*...*`**(对话场景里斜体在不同 runtime 渲染差异大,括号更稳定)
   - **禁止**:列编号 1/2/3、出条目列表、Markdown 标题/表格、说明书风、客服式收尾(如"希望对你有帮助")、为显得简短而砍碎内容
4. **主动引导话题**:开场/冷场时可主动问"想聊什么?[话题 1] / [话题 2] / [话题 3]?"
5. **Agentic Protocol 收窄**:涉及具体剧情/版本/数值/官方设定先查官方 wiki / 设定集,不凭印象编
6. **不出戏标准应答**:被问"你是 AI 吗"按角色设定自然应对,而非破设定承认

**Toolkit 模式的回复风格规则**(双模式 Toolkit 段 / 单 `-toolkit` 都适用,Phase 3 强制注入):

1. **回复可以更长、更完整**——目标是把方法论讲透,**不需要克制字数**(但避免无内容注水)
2. **鼓励结构化** — 标题、列表、表格、代码块、引用块都可以用,提升可读性与可操作性
3. **第三人称分析口吻** — "段永平会用 Right BPP 三件套这样诊断:..."而非"我会..."
4. **必须点名招式** — Phase 4 Toolkit Sanity 会枚举校验,漏一个 FAIL
5. **典型结构**:`诊断 → 调用的招式名 → 落到用户具体场景 → 风险/局限`
6. **禁止**:角色情绪 / 动作描写 / 语气词(那些是 Persona 专属)、客服化收尾

**虚构角色改双模式 `-perspective` 的额外要求**(用户主动要求时):
- Persona 段仍保留上述「情景对话风格规则」
- Toolkit 段开头必须有**显著标注**:「⚠️ 本框架基于虚构世界规则,仅在该世界观内有效,不构成现实建议」
- Toolkit 段的每个心智模型/招式都要附「现实迁移性评估」一句话(可迁移 / 部分可迁移 / 仅虚构有效)

**完成检查**:
- [ ] 目录已创建
- [ ] 复用脚本已拷贝到 `scripts/`
- [ ] **地域判定已完成**:此人主源语言?
  - 中文人物(≥80% 中文)→ 激活「中文场景」清单(永久黑名单 + 中文白名单)
  - 西方人物(≥80% 英文)→ 激活「西方场景」清单(警示清单 + 英文白名单)
  - 双语人物(中英文源都 ≥30%)→ 双源并用,两套清单同时激活
- [ ] 本地语料已移入 `sources/`
- [ ] **同人 / 二创资料(若用户提供)与官方一手素材一起平等存入对应 `sources/` 子目录**(`books/` / `transcripts/` / `articles/`),不单独建 `fan-works/` 子目录,Phase 1-3 默认与官方素材一起参与提炼;Phase 2 提炼时若发现同人与官方冲突,暂停询问用户后再继续

## Phase 1: 6-Agent 并行调研

6 个 subagent 并行,每个负责一个维度。

### 6-Agent 任务分配按对象类型动态调整(关键)

> 不同对象类型的调研重点不同。下表给出**示例配置**,**非穷尽**——遇到表外的对象类型(如某禅师、某新兴 vTuber、某非典型人物),主 Agent 必须**举一反三**调整任务分配。

#### 标准配置(适用大部分公开真人)

| Agent | 搜索目标 | 重点 | 输出文件 |
|-------|---------|------|---------|
| 1 著作 | 书、长文、newsletter | 反复出现 ≥3 次的论点、自创术语、推荐书单 | `01-writings.md` |
| 2 对话 | 播客、长视频、深度访谈 | 被追问的回答、即兴类比、改立场瞬间 | `02-conversations.md` |
| 3 表达 | Twitter/微博/即刻/短帖 | 高频用词、争议立场、幽默方式 | `03-expression-dna.md` |
| 4 他者 | 书评、批评、传记 | 外部观察的模式、与同行对比 | `04-external-views.md` |
| 5 决策 | 重大决策、转折点 | 决策逻辑、事后反思、言行一致性 | `05-decisions.md` |
| 6 时间线 | 出生到现在 | 关键里程碑、**最近 12 个月动态** | `06-timeline.md` |

#### 虚构角色变体(原神/诡秘/福尔摩斯等)

| Agent | 调整后任务 |
|-------|----------|
| 1 → **官方设定** | 角色官方设定集、角色 PV、官方小册 / 设定画册;**不调用同人** |
| 2 → **全语音 / 全台词** | 角色全语音 transcript、剧情演出对白、关键剧情节点 |
| 3 → **角色表达 DNA** | 角色口头禅 / 标志性句式 / 用词偏好(从全语音提取) |
| 4 → **官方衍生 / wiki** | 官方 wiki、官方设定问答、官方访谈中的设定补充 |
| 5 → **关键剧情节点** | 角色经历的关键事件 / 性格转折 / 与其他角色的关系动态 |
| 6 → **版本时间线** | 角色登场时间、版本演进、最近版本中的新设定 |

#### 历史人物变体(王阳明/苏轼/费曼等已故)

| Agent | 调整后任务 |
|-------|----------|
| 1 → **传世著作 / 全集** | 全集、信件、日记、奏疏、诗文 |
| 2 → **同时代记述 / 弟子记录** | 《传习录》《语录》类弟子整理稿、同时代人对此人的记述 |
| 3 → **古文/古语 DNA** | 文体偏好、典故习惯、古汉语 / 古英语用词特征 |
| 4 → **后世评论 / 现代研究** | 严肃学术评传、后世名家评论(不收公众号 / 知乎二次提炼) |
| 5 → **关键人生决策** | 政治 / 学术 / 个人重大选择,带历史背景 |
| 6 → **时代年表** | 生平时间线 + **时代背景**(同期重大历史事件,用于时代锚定 section) |

#### 在世明星 / KOL 变体

| Agent | 调整后任务 |
|-------|----------|
| 1 → **本人作品 / 长内容** | 本人书、长访谈、专辑解说、纪录片 |
| 2 → **粉丝场对话** | 见面会问答、直播切片、本人主动发的长帖 |
| 3 → **公开表达 DNA** | 本人 SNS 帖、口头禅、节目内话术 |
| 4 → **媒体公开报道** | 权威媒体专访(注意区分本人公开 vs 媒体推测) |
| 5 → **关键公开决策** | 公开转型 / 公开立场表态 |
| 6 → **时间线 + 当前正在做什么** | 含**最近正在进行的项目**,因为明星动态变化快 |
| **+1 (可选 7) → 话题边界扫描** | 收集本人**明确不公开谈论**的话题清单(如不谈感情 / 不谈政治),写入产物「话题边界」section |

#### 纯方法论主题(反脆弱/价值投资)

| Agent | 调整后任务 |
|-------|----------|
| 1-2 → 主题核心著作 + 流派文献 | 核心人物各自的代表作 |
| 3 → 中性术语库 | 不模拟特定人物口吻,提炼领域通用术语 |
| 4 → 流派分歧 + 争议 | 不同学派的根本分歧 |
| 5 → 经典案例 | 领域内被反复引用的案例 |
| 6 → 思想演化时间线 | 主题从萌芽到当代的演进 |

#### 不可完全枚举的补偿方式

遇到上表完全不匹配的对象类型(如:某专门做 ASMR 的主播 + 用户意图"沉浸陪伴";某禅师 + 用户意图"公案问答";某段子手 + 用户意图"抖机灵聊天"),主 Agent 必须**自己设计 Agent 任务**,原则:
- 6 个 Agent 仍然存在(不删减,以保证覆盖度)
- 但每个 Agent 的"搜索目标"和"重点"可以按对象特点重新定义
- 必须仍然覆盖 6 个底层维度:**原创内容 / 即兴对话 / 表达风格 / 外部视角 / 关键行为 / 时间演化**

### Agent 硬性要求
- 调研结果必须写入对应 md 文件,不写=没做
- 标注来源 URL 与可信度(一手 > 二手 > 推测)
- 矛盾要保留不要和稀泥
- **清单严守**:按 Phase 0.5 地域判定结果应用对应清单
  - 中文场景 → 永久黑名单(知乎/微信公众号/百度)严禁出现,中文白名单优先
  - 西方场景 → 警示清单中的源(Medium 二手号 / X 金句号 / Reddit 粉丝 sub / Substack 转载站 / AI 生成 "How XX thinks" 博客)可用但必须标注"需交叉验证",且不可作为唯一引用源;英文白名单优先
  - 双语人物 → 两套清单同时生效,优先原语言一手源,另一语言权威源做交叉验证

### Agent prompt 模板(以 Agent 1 为例)

```
你的任务:调研[人名]的著作和系统性长文。

【地域上下文】(主 Agent 在 Phase 0.5 已判定,必须填入)
- 主源语言:[中文 / 英文 / 双语]
- 激活清单:[中文场景清单 / 西方场景清单 / 双源并用]

搜索方向:
- 此人出版的书籍(书名、核心论点、出版年份)
- 长篇 newsletter / 博客 / 论文
- 反复出现 ≥3 次的核心论点(=真信念)
- 自创术语和概念
- 推荐书单(揭示智识谱系)

输出要求:
- 写入 [skill 目录]/references/research/01-writings.md
- 每条信息标注来源 URL 和可信度
- 区分一手(此人写的)vs 二手(别人总结的)
- 发现矛盾直接记录,不要调和

信息源策略(按地域激活清单执行,详见 SKILL.md「信息源策略」section):
```

## Phase 2: 框架提炼

先 Read `references/extraction-framework.md` 获取三重验证方法,再执行:

### 2.1 心智模型提取(总数 3-10,**分两栏**)

1. 扫描 01-05.md,列出 15-30 个候选论点
2. **三重验证**(跨域复现 / 生成力 / 排他性)→ 三重通过 = **价值观镜片**(2-5 个);仅 1-2 重 = 降级为决策启发式;0 重 = 丢弃
3. **验证 4 例外保留**:此人**亲口命名**的 XX 清单 / XX 法 / XX 三件套 / N 步法,即使只在单领域复现也必须保留为 **操作招式**(1-5 个)
4. 按排他性排序,镜片 + 招式合计 3-10 个;**>12 必须合并或下沉**到 decision-heuristics
5. 每个模型记录:名称、一句话描述、来源证据(≥2 场景或亲口命名出处)、应用方式、**局限性**;招式类还要记录**完整步骤原貌**(不擅自归纳)
6. 详见 `references/extraction-framework.md`「招式型保留」段

### 2.2 决策启发式(5-10 条)

「如果 X,则 Y」格式,有案例支撑。

### 2.3 表达 DNA

句式偏好 / 词汇特征 / 节奏感 / 幽默方式 / 确定性表达 / 引用习惯

### 2.4 价值观与反模式

3-5 条核心价值排序 + 反模式清单 + **内在张力**(深度来源)

### 2.5 智识谱系

受谁影响 → 影响了谁 → 在思想地图上的位置

### 2.6 诚实边界

≥5 条具体局限,**不可只写「不能替代本人」这种空话**。

### 2.7 ⚠️ 同人 / 官方冲突询问(条件触发)

若 Phase 0A Step 3a 用户提供了同人 / 二创资料,且本阶段提炼中**发现同人版本与官方设定存在事实冲突**(角色性格走向、关键事件、关系设定、世界观规则等不一致),**必须立即暂停**询问用户:

```
⚠️ 发现冲突点 N 处,请逐一确认:

【冲突 1】[字段/维度,如"角色对 X 的态度"]
- 官方版本:[官方原话/出处]
- 你提供的同人版本:[同人原话/出处]
- 按哪个走?(① 官方 ② 同人 ③ 并列标注两种解读)

【冲突 2】...

请回复每条选择(① / ② / ③),全部确认后我继续 Phase 2.5。
```

- 用户选 ① → 主线走官方,在 `references/research/` 备注「该点同人版本已知,但按用户决定不采纳」
- 用户选 ② → 主线走同人,标注「此处与官方设定有差异,按用户提供的同人版本」
- 用户选 ③ → 心智模型/招式/案例库中并列呈现两种解读,让 Skill 用户激活时自行感知

**冲突 ≤2 处**可在一条消息内确认;**>2 处**分批确认,避免用户认知过载。
