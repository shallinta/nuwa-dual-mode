# Toolkit 工作流子模板(按意图分类)

> **核心原则**:Toolkit 不是只有"4 步诊断"一种工作流。不同意图需要不同的工作流。本文件列出**示例子模板**,**非穷尽**——遇到不适合任何示例的人物 / 意图,主 Agent 必须**举一反三**设计新的合适工作流并写入产物 SKILL.md。

---

## 何时调用本文件

Phase 3 构建 SKILL.md 时,根据 Phase 0A 推断出的**意图维度列表**,为每个 Toolkit 倾向的意图选用 / 设计对应工作流。

| 推断意图 | 推荐子模板 |
|---------|----------|
| 方法论学习 / 决策辅助 | A. 诊断型 |
| 创作指导(写作/视频/设计) | B. 作品 Review 型 |
| 人生哲学借鉴 | C. 思想借鉴型 |
| 精神镜片(自我审视) | D. 价值观对照型 |
| 专业领域咨询(伪) | E. 案例分析型(必带免责) |
| 决策复盘陪练 | F. 苏格拉底追问型 |
| 学术辩论 | G. 立场反驳型 |
| **其他** | **举一反三,新建子模板写入产物** |

> ⚠️ 本表仅列出 **Toolkit 倾向**的意图维度。Persona 倾向的意图(角色扮演对话 / 情感陪伴 / 风格模仿创作 / 古风语境玩味等)走 Persona 模式,不在此表覆盖范围,其工作流由 `dual-mode-template.md` / `single-mode-template.md` 的 Persona section 定义。

---

## 子模板 A:诊断型(默认,适用大部分公开方法论人物)

**适用**:投资家 / 企业家 / 学者 / 任何有可执行框架的人

**4 步**:
1. **诊断**:锁定问题落在哪个心智模型 / 招式
2. **追问**:用此人钩子问题深挖(1-2 个,不堆砌)
3. **建议**:四段式输出 `视角诊断 → 调用招式 → 落到用户场景 → 风险/局限`
4. **警惕**:触碰反模式时明确指出,带温度不评判

**示例人物**:段永平、芒格、巴菲特、塔勒布、张磊

---

## 子模板 B:作品 Review 型

**适用**:作家 / 编剧 / 视频创作者 / 设计师 / 摄影师

**4 步**:
1. **理解**:先用一句话复述用户作品的核心(创意 / 故事 / 视觉 / 体验)。误读则不评
2. **诊断**:用此人创作方法论扫描——结构 / 节奏 / 信息密度 / 钩子 / 风险点
3. **改稿建议**:不只是"不好",给出**具体改法**(改字 / 改结构 / 删减 / 重组),最好附"改后示例片段"
4. **本人风格借鉴**:点名"如果是 [人名] 来写,他会怎么处理这一段"——必须召回此人亲口讲过的招式

**示例人物**:爱潜水的乌贼(网文世界观构建法)、村上春树(短句节奏)、MrBeast(注意力工程)、原研哉(设计减法)

---

## 子模板 C:思想借鉴型

**适用**:哲学家 / 思想家 / 宗教人物 / 古典学者

**3 步**:
1. **场景拆解**:把用户问题翻译成此人思想体系内的核心矛盾(用此人术语,如王阳明的"知行" / 罗翔的"自我感动")
2. **思想引用**:援引此人**亲口说过**或**著作里写过**的原句 / 段落,标明出处
3. **当代对应**:把古意 / 抽象观念翻译到用户的具体处境,**不强行套用**,明确说"如果不适用,请只取启发"

**示例人物**:王阳明、孔子、苏格拉底、罗翔、刘擎、稻盛和夫

---

## 子模板 D:价值观对照型(精神镜片)

**适用**:任何价值观稳定鲜明的人(适合作为道德 / 审美参照)

**3 步**:
1. **现状镜照**:用此人价值观核心维度(例如:稻盛和夫的"作为人何为正确" / 段永平的"本分")扫描用户当前选择
2. **温和评估**:此人会**赞同 / 中性 / 不赞同**用户的哪些部分,**用此人会用的语气**(不评判,只指出)
3. **没有结论**:不替用户决定,只把镜子举起来

**关键**:这个子模板**禁止**说"你应该 X",只能说"如果按 [人名] 的价值观看,这一面是 X,另一面是 Y"

**示例人物**:稻盛和夫、王阳明、史怀哲、特蕾莎修女、罗翔

---

## 子模板 E:案例分析型(专业领域,必带免责)

**适用**:明星医生 / 明星律师 / 明星投资人 / 明星心理师

**4 步**:
1. **强制免责**:首句必出现"本 Skill 不构成 [医疗/法律/投资/心理] 建议,严重情况请立即咨询真实专业人士并拨打官方求助热线"
2. **教科书分析**:用此人**公开发表过**的方法论 / 案例分析框架,把用户问题作为案例做学理分析
3. **场景类比**:引用此人公开讲过的类似案例,告诉用户"这类问题历史上一般怎么发展"
4. **再次免责 + 转介**:结尾再提醒一次,并建议用户找真实专业人士

**示例人物**:罗翔(刑法案例)、张文宏(公共卫生科普)、武志红(心理观察)

**红线**:**不可**给出具体诊断 / 具体药品 / 具体法律意见 / 具体投资标的

## 子模板 F:苏格拉底追问型(决策复盘陪练)

**适用**:善于钩子问题的人(芒格、苏格拉底、费曼、稻盛和夫)

**对话循环**:
1. **接收**:听用户讲一遍他的决策 / 想法
2. **追问 1**:用此人最标志性的钩子问题之一反追(每轮只问 1 个)
3. 用户答 → **追问 2**(基于答案深一层)
4. 反复 3-5 轮,直到用户**自己**讲出新东西
5. **不下结论**:最后只回应"看起来你已经想清楚 X,但 Y 还没收敛——下次自己再走一遍"

**关键**:这个子模板**不给答案**,只问问题。Skill 必须列出**至少 10 个**此人亲口问过的钩子问题作为弹药库。

**示例人物**:芒格、苏格拉底、费曼、稻盛和夫(六项精进)、彼得德鲁克

---

## 子模板 G:立场反驳型(学术辩论)

**适用**:立场鲜明 / 善于论辩的人

**3 步**:
1. **接受立场**:复述用户立场,确认无误解
2. **此人式反驳**:用此人**公开持有的反向立场**(或最可能持有的反向立场)逐点反驳,每点带:本人原话 / 案例 / 推理
3. **不强求胜利**:结尾说"如果你的立场能扛住以上反驳,那它就比之前更稳"

**关键**:反驳必须基于此人**真实立场**,不能编造一个"假反对"。如果此人没有反向立场,这个子模板就不适用。

**示例人物**:塔勒布(反学院派)、刘擎(对反智立场)、罗翔(对功利主义)、Naval(对工业化教育)

## 举一反三:遇到不适配示例的情况

如果 Phase 0A 推断出的意图维度不属于以上 A-G,**主 Agent 必须自己设计子模板**写入产物 SKILL.md。设计原则:

1. **必须 3-5 步**(短了不够操作,长了用户懒得读)
2. **每一步必须有具体动作**(不能写"分析问题"这种空话)
3. **必须明确"什么时候触发"**(用户用什么样的话激活这个子模板)
4. **必须召回此人亲口招式**(不能输出泛化建议)
5. **如果涉及敏感领域,必须带免责**(参照子模板 E)

例如:
- 蒸馏一个游戏 UP 主 + 意图"装备搭配推荐" → 设计「装备评分 → 流派分类 → 推荐组合 → 升级路径」4 步
- 蒸馏一个段子手 + 意图"抖机灵聊天" → 这个其实是 Persona 不是 Toolkit,跳过
- 蒸馏一个禅师 + 意图"公案问答" → 设计「先抛公案 → 听用户解 → 不直接评对错 → 再抛一个反向公案」循环

---

## 写入产物 SKILL.md 的方式

每个选用 / 新建的子模板,在产物 SKILL.md 的 Toolkit 段以**一段简短概述 + 一行"详见 references/.../diagnostic-workflow-[A/B/C/...].md"** 的方式呈现。子模板的详写下沉到 `mode-toolkit/` 子目录的对应文件,主文件保持 <200 行。
